# 토스 개발자 센터 로그인 설정 가이드

이 가이드는 토스 개발자 센터에서 Living Craft 앱의 토스 로그인 기능을 설정하는 방법을 단계별로 안내합니다.

---

## 📋 사전 준비 사항

✅ **완료한 작업:**
- 사업자등록증 제출 완료
- 이용약관 2개 작성 완료 (`living-craft-terms.md`, `living-craft-privacy.md`)

✅ **필요한 것:**
- GitHub Gist에 업로드한 약관 URL 2개
- webhook.site에서 생성한 임시 콜백 URL (또는 실제 서버 URL)

---

## 🚀 설정 단계

### Step 1: 토스 개발자 센터 접속

1. https://developers-apps-in-toss.toss.im/ 접속
2. **대표 관리자 계정**으로 로그인 (⚠️ 대표 관리자만 설정 가능)
3. 워크스페이스 선택
4. Living Craft 미니앱 선택
5. 좌측 메뉴에서 **"토스 로그인"** 클릭

---

### Step 2: 약관 확인 및 동의

1. "약관 확인하기" 버튼 클릭
2. 토스 로그인 서비스 약관 읽기
3. 동의 체크박스 선택
4. "동의하기" 버튼 클릭

---

### Step 3: 동의 항목 선택

**Living Craft에 필요한 항목:**

| 항목 | 스코프 이름 | 선택 | 이유 |
|------|------------|------|------|
| ✅ 이름 | `USER_NAME` | **필수** | 예약자 신원 확인 |
| ✅ 전화번호 | `USER_PHONE` | **필수** | 예약 확인 및 연락 |
| ⬜ 이메일 | `USER_EMAIL` | 선택 | 알림용 (추후 추가 가능) |
| ⬜ CI | `USER_CI` | 선택 | 중복 가입 방지 (추후 추가 가능) |

#### 선택 방법:
1. 콘솔에서 "② 동의 항목 선택" 섹션 찾기
2. **이름** 체크박스 선택
3. **전화번호** 체크박스 선택

⚠️ **중요:** 전화번호를 선택했으므로 **연결 끊기 콜백 정보**를 반드시 입력해야 합니다!

---

### Step 4: 이용약관 URL 등록

콘솔의 "③ 약관 등록" 섹션에서 설정합니다.

#### 자동 포함되는 약관 (입력 불필요)
- ✅ 토스 로그인 서비스 약관
- ✅ 개인정보 제3자 제공 동의

#### 직접 등록해야 하는 약관

**1. 서비스 이용약관 (필수)**
- 약관 이름: `Living Craft 서비스 이용약관`
- 약관 URL: `https://gist.githubusercontent.com/[your-username]/[gist-id]/raw/living-craft-terms.md`
- ✅ 필수 체크박스 선택

**2. 개인정보 수집·이용 동의 (필수)**
- 약관 이름: `Living Craft 개인정보 수집·이용 동의`
- 약관 URL: `https://gist.githubusercontent.com/[your-username]/[gist-id]/raw/living-craft-privacy.md`
- ✅ 필수 체크박스 선택

**3. 마케팅 정보 수신 동의 (선택사항, 추후 추가 가능)**
- 나중에 필요하면 추가

#### 입력 예시:
```
서비스 이용약관
┌────────────────────────────────────────────────────────────┐
│ 약관 이름: Living Craft 서비스 이용약관                      │
│ 약관 URL: https://gist.githubusercontent.com/...          │
│ ☑ 필수                                                      │
└────────────────────────────────────────────────────────────┘

개인정보 수집·이용 동의
┌────────────────────────────────────────────────────────────┐
│ 약관 이름: Living Craft 개인정보 수집·이용 동의             │
│ 약관 URL: https://gist.githubusercontent.com/...          │
│ ☑ 필수                                                      │
└────────────────────────────────────────────────────────────┘
```

⚠️ **확인사항:**
- URL을 브라우저에서 열어보고 약관이 정상적으로 보이는지 확인
- URL에 오타가 없는지 확인
- 약관 내용이 화면에 명확하게 노출되는지 확인

---

### Step 5: 연결 끊기 콜백 URL 설정

콘솔의 "④ 연결 끊기 콜백 정보" 섹션에서 설정합니다.

#### 로컬 개발 단계 (임시 설정)

**Option A: webhook.site 사용 (권장)**

1. https://webhook.site/ 접속
2. 자동 생성된 고유 URL 복사 (예: `https://webhook.site/abc123...`)
3. 토스 콘솔에 입력:

```
연결 끊기 콜백 정보
┌────────────────────────────────────────────────────────────┐
│ 콜백 URL: https://webhook.site/abc123...                   │
│ HTTP 메서드: POST ▼                                         │
│ Basic Auth 헤더: (비워둠)                                    │
└────────────────────────────────────────────────────────────┘
```

**Option B: 실제 서버 URL (프로덕션 준비 완료 시)**

```
연결 끊기 콜백 정보
┌────────────────────────────────────────────────────────────┐
│ 콜백 URL: https://api.living-craft.com/api/auth/toss-unlink│
│ HTTP 메서드: POST ▼                                         │
│ Basic Auth 헤더: username:password (선택)                   │
└────────────────────────────────────────────────────────────┘
```

#### 연결 끊기 이벤트 종류

콜백 호출 시 `referrer` 파라미터로 구분됩니다:

| referrer | 설명 | Backend 처리 |
|----------|------|--------------|
| `UNLINK` | 사용자가 앱에서 직접 연결 끊기 | 로그아웃 처리 |
| `WITHDRAWAL_TERMS` | 로그인 서비스 약관 철회 | 사용자 정보 삭제 권장 |
| `WITHDRAWAL_TOSS` | 토스 회원 탈퇴 | 사용자 정보 삭제 필수 |

---

### Step 6: 복호화 키 수신

설정 완료 후 콘솔에서 **"복호화 키 확인하기"** 섹션을 확인합니다.

1. **"이메일로 복호화 키 받기"** 버튼 클릭
2. 등록된 이메일 확인
3. 이메일로 수신한 **AES 256비트 복호화 키** 안전하게 보관

⚠️ **보안 주의사항:**
- 절대 외부에 노출 금지
- GitHub, 공개 저장소에 커밋 금지
- 환경 변수(`.env`)로 관리
- 재발급이 필요하면 채널톡으로 문의

#### 복호화 키 사용처:
- 토스 API에서 받은 **암호화된 사용자 정보**(이름, 전화번호)를 복호화할 때 사용
- Backend에서만 사용 (Frontend에 절대 노출 금지)

---

### Step 7: 설정 완료 및 확인

1. 모든 입력 사항 재확인
2. "저장" 또는 "완료" 버튼 클릭
3. 설정 완료 메시지 확인

---

## 📧 완료 후 확인 사항

설정 완료 후 다음 정보가 준비되어야 합니다:

| 항목 | 확인 방법 | 보관 위치 |
|------|----------|----------|
| ✅ 복호화 키 | 이메일 수신 | Backend `.env` 파일 |
| ✅ 서비스 이용약관 URL | GitHub Gist Raw URL | 토스 콘솔에 등록됨 |
| ✅ 개인정보 수집·이용 동의 URL | GitHub Gist Raw URL | 토스 콘솔에 등록됨 |
| ✅ 연결 끊기 콜백 URL | webhook.site 또는 실제 URL | 토스 콘솔에 등록됨 |

## 🔧 문제 해결

### Q1: "대표 관리자만 설정할 수 있습니다" 오류

**원인:** 현재 계정이 대표 관리자가 아님

**해결:**
1. 토스 개발자 센터에서 로그아웃
2. 워크스페이스의 **대표 관리자 계정**으로 로그인
3. 다시 시도

---

### Q2: 약관 URL이 접속되지 않아요

**원인:** GitHub Gist가 Private으로 생성됨

**해결:**
1. GitHub Gist 페이지로 이동
2. 우측 상단 "Make public" 버튼 클릭
3. 또는 삭제 후 "Create public gist"로 재생성

---

### Q3: 복호화 키 이메일이 오지 않아요

**가능한 원인:**
- 스팸 메일함 확인
- 등록된 이메일 주소 오류
- 토스 시스템 지연

**해결:**
1. 스팸 메일함 확인
2. 10분 대기 후 재시도
3. 여전히 안 오면 채널톡으로 문의

---

### Q4: client_id와 client_secret은 어디서 확인하나요?

**답변:**
Apps-in-Toss 토스 로그인은 **복호화 키** 방식을 사용하며, 별도의 client_id/client_secret이 발급되지 않습니다.

다음 정보만 있으면 됩니다:
- ✅ 복호화 키 (이메일로 수신)
- ✅ 토스 API 엔드포인트 (문서에 명시됨)

---

### Q5: CONSOLE_API_KEY는 무엇인가요?

**답변:**
`.env` 파일의 `CONSOLE_API_KEY`는 토스 개발자 콘솔 API 키이며, **토스 로그인과는 별개**입니다.

- `CONSOLE_API_KEY`: 콘솔 API 접근용 (앱 관리)
- `복호화 키`: 토스 로그인 사용자 정보 복호화용
