# PRD — 가구 자산 증감 트래커 (가칭: 자산일기)

> **산출물**: 본 문서를 마크다운 파일로 정리하여 Claude Design 에 입력, 화면 프로토타입 생성에 활용합니다.
> **플랫폼**: Apps-in-Toss Mini App (Granite + TDS React Native)
> **대상**: 1개 가구 워크스페이스를 다수 멤버(편집자/뷰어)가 공유하는 형태. 부부 + 자녀/부모 등 확장 가능.

---

## 1. Context (왜 만드는가)

기존 가계부/자산관리 앱은 다음 두 영역 중 하나만 잘 합니다.

- **가계부 앱**: 일자별 수입/지출 흐름은 잘 보여주지만, "내 전체 자산이 작년 대비 몇 % 변했는가" 는 알 수 없음.
- **자산관리 앱**: 자산 평가는 해주지만, 그 변화가 어떤 현금흐름·거래에서 비롯됐는지 맥락이 약함.

문제: **"내 주식이 100% 올랐다"는 사실은 알아도, 그게 내 전체 가구 순자산 기준 몇 %인지** 직관적으로 보기 어렵습니다.
주식 비중이 1%라면 100% 수익도 가구 자산엔 +1% 영향에 그칩니다. 이 비중·기여도를 사용자가 직접 계산하기는 매우 번거롭습니다.

목표: 사용자가 **원하는 시점에 자산 스냅샷과 거래 내역**을 손쉽게 기록하면, 자동으로
1. **연간 가구 순자산 증감률** (전년 대비 % / 절대액)
2. **자산군별 기여도** (어떤 자산이 얼마나 끌어올렸는가)
3. **현금흐름 요약** (수입·지출 카테고리, 저축률)

을 보여주는 1가구용 자산일기 미니앱을 만듭니다. 가구 단위 워크스페이스를 만들고, 가족 구성원을 권한별로 초대하여 함께 기록·조회합니다. 본 앱이 다루는 자산은 **가구 공동 자산**으로 한정합니다(개인 통장/개인 자산은 각자가 별도 관리하는 것으로 가정하고 범위 제외).

## 2. 핵심 가치 제안

| 기존 | 자산일기 |
|---|---|
| 종목별 수익률 ↑ | **가구 순자산 증감률 ↑** |
| 매일 자동 동기화 부담 | **원할 때 스냅샷 (가벼운 입력)** |
| 가계부 ≠ 자산관리 | **거래 + 자산 스냅샷 통합** |
| 개인 단위 | **가구 단위, 멤버 권한 분리 공유** |
| 매월 수동 반복 입력 | **고정지출 자동 등록 (구독·월세·공과금)** |

## 3. 페르소나

**P1. 가구 오너 (1차 페르소나)**
- 30대 직장인, 재테크 관심 중상
- 주식·코인·예금 계좌 분산, 매번 합산 계산이 귀찮음
- "올해 우리 집 자산 얼마나 늘었지?" 가 가장 궁금
- 매일 입력 = 부담, 월 1회 정도가 적정
- 가구 워크스페이스 생성자, 멤버 초대·권한 부여 권한 보유

**P2. 배우자 (편집자)**
- 가구 공동 자산·거래를 함께 기록·수정
- 입력 빈도 P1과 비슷하거나 낮음
- 자녀 양육비·생활비 등 직접 입력하는 시나리오

**P3. 자녀/부모 (뷰어, 선택적)**
- 가구 자산을 조회만 가능
- 부모님께 자산 현황 공유, 또는 성인 자녀가 부모 자산을 도와 모니터링하는 시나리오
- 편집·삭제·초대 권한 없음

## 4. 핵심 기능 (MVP)

### 4.1 자산 관리
사용자는 자산을 등록하고, 원하는 시점마다 **평가액 스냅샷**을 기록합니다.

**자산 카테고리 (1차)**
- **현금성**: 입출금 통장, 예적금, 청약
- **투자**: 국내/해외 주식, ETF, 펀드, 채권 (종목 단위 또는 계좌 단위 자유 선택)
- **가상자산**: 코인 (지갑 또는 거래소 계좌 단위)
- **실물자산**: 부동산, 자동차, 금/은 (스냅샷 방식 — 사용자가 시세 수동 기입)
- **부채**: 주담대, 신용대출, 카드빚 (마이너스 자산)
- **연금/보험**: 국민연금, 퇴직연금, 저축성 보험

**자산 데이터 모델**
```
Asset {
  id
  name              // "삼성전자", "신한 주거래", "강남 아파트", "미래에셋 미국주식"
  category          // 위 6개 중 하나
  currency          // KRW (기본) | USD | EUR | JPY | ... (옵션)
  isLiability       // 부채 여부 (true면 음수로 합산)
  memo              // 자유 메모
  createdAt
  archivedAt        // 처분/해지 시 비활성화
}

AssetSnapshot {
  id
  assetId
  date              // 사용자가 지정한 평가 시점
  value             // 해당 시점 평가액 (자산의 currency 단위)
  fxRateToKRW       // 1 단위당 KRW 환율. currency=KRW 면 1 또는 null
  valueKRW          // value × fxRateToKRW (저장값, 합산용 캐시)
  note              // "오늘 종가 기준", "감정평가서 받음" 등
}
```

**가격 변경 추적**: 동일 자산에 여러 스냅샷이 쌓이면 자동으로 시계열 그래프 생성.
**평가 시점 규칙**: 특정 날짜 기준 자산 합산 시, 해당 자산의 **그 날짜 이전 가장 최근 스냅샷** 사용.

### 4.1.1 다통화 지원 (Optional per Asset)

해외 주식·외화 예금 등 **외화 잔액으로 보관되는 자산**을 원화로 환산하지 않고 본 통화 단위로 기록할 수 있습니다.

**유스 케이스**
- 미래에셋 미국 주식 계좌: 5,000 USD 보유
- 신한 외화 예금: 3,000 USD
- 일본 여행용 엔화 통장: 50,000 JPY

**동작 방식**
- 자산 등록 시 통화 선택 (default: KRW). 한 번 정한 통화는 자산 단위로 고정.
- 스냅샷 입력 시 **본 통화의 금액 + 그 시점 환율**을 함께 입력 (예: 5,000 USD / 1 USD = 1,380 KRW)
- 환율은 **수동 입력**이 기본 (사용자가 검색해서 입력). 시세 자동 조회는 Phase 2.
- 자산 상세 화면: `5,000 USD ≈ 6,900,000원` 형태로 본 통화 + 원화 환산을 함께 표기
- 대시보드/현금흐름의 모든 합산은 **KRW 환산값(`valueKRW`)** 기준
- 자산군별 기여도 분석 시, 해당 외화 자산의 변동에는 **본 통화 평가액 변동 + 환율 변동** 두 요인이 모두 반영됨 (Phase 2에서 평가손익 / 환차손익 분리 분석 검토)

**거래(Transaction) 통화는 KRW 고정 (MVP)**
- 외화 자산의 매수/매도/송금 등 외화 거래는 MVP 에서는 **KRW 환산 금액으로 입력**합니다 (편의성 우선).
- 외화 거래의 본 통화 단위 추적과 환차손익 자동 계산은 Phase 2 후보.

### 4.2 거래 (용돈기입장)
사용자가 원하는 시점에 자유롭게 기록하는 수입·지출 내역.

**거래 데이터 모델**
```
Transaction {
  id
  date              // 거래일
  type              // INCOME | EXPENSE | TRANSFER
  amount            // 원
  categoryId        // 사용자 정의 카테고리
  fromAssetId       // 출금 자산 (지출/이체 시)
  toAssetId         // 입금 자산 (수입/이체 시)
  title             // "1월 전기세"
  memo              // "사용량 320kWh, 평소보다 +50"
  attachments       // 영수증 사진 (선택)
  tags              // 자유 태그
}
```

**카테고리 시스템**
- **수입**: 급여, 사업소득, 투자수익, 기타
- **지출**: 주거(공과금/관리비/월세), 식비, 교통, 의료, 쇼핑, 여가, 교육, 보험료, 구독, 기타
- **이체**: 자산 간 이동 (수익/지출 아님, 합산 영향 없음)
- 사용자 커스텀 카테고리 추가 가능

**부수 메모 활용 예시**
- 공과금 → "전기 320kWh / 가스 18㎥ / 수도 12톤"
- 차량 주유 → "주유 50L / 누적 km"
- → 단순 금액 외 사용량/단가 정보를 자유 텍스트로 누적

### 4.2.1 정기/고정 지출 (RecurringTransaction)

매월 또는 매년 동일하게 발생하는 거래를 **템플릿으로 등록**해두면, 지정한 일자에 자동으로 거래 내역이 생성됩니다.

**유스 케이스**
- 구독 서비스: 넷플릭스 매월 17일 13,500원
- 월세 / 관리비: 매월 1일 출금
- 공과금: 매월 25일 (실 사용 금액은 템플릿 생성 후 수정)
- 보험료: 매월 5일 자동이체
- 자동차 보험: 매년 갱신일

**데이터 모델**
```
RecurringTransaction {
  id
  type              // INCOME | EXPENSE | TRANSFER
  amount            // 기본 금액 (수정 가능)
  categoryId
  fromAssetId
  toAssetId
  title             // "넷플릭스 구독"
  memo
  schedule {
    frequency       // MONTHLY | YEARLY
    dayOfMonth      // 1~31, 또는 'LAST' (말일)
    monthOfYear     // YEARLY 일 때만
  }
  startDate         // 적용 시작일
  endDate           // 종료일 (null = 무기한)
  active            // 일시 중지 가능
  generatedTxIds    // 이 템플릿에서 생성된 Transaction 목록
}
```

**자동 생성 규칙**
- 매일 새벽 1회 스케줄 검사 → 오늘이 `dayOfMonth` 인 활성 템플릿 → `Transaction` 생성
- 생성된 거래는 **"자동 생성됨"** 뱃지로 표시, 사용자가 금액·메모 수정 가능 (예: 매월 다른 공과금 금액 반영)
- 같은 템플릿이 같은 날짜에 중복 생성되지 않도록 멱등성 보장
- 월말 처리: `dayOfMonth=31` 인데 2월이라면 해당 월 말일에 생성

**알림**
- 자동 거래 생성 시 가구 멤버(편집자 이상)에게 토스 푸시 알림 (선택)
- 카드/계좌 잔액 부족 예측 시 사전 알림은 Phase 2

### 4.3 가구 순자산 대시보드 (메인 화면)

**상단: 현재 가구 순자산**
- 총자산 - 총부채 = 순자산 (큰 숫자)
- 전년 동기 대비 **% 증감**, 절대액 증감 (강조 색상)
- 직전 스냅샷일자 표시 ("3일 전 업데이트")

**중단: 연간 자산 변화 그래프**
- X축: 월, Y축: 순자산
- 토글: 1년 / 3년 / 전체
- 라인 위 마커는 스냅샷 입력일

**하단: 자산군별 기여도 (이번 해)**
- 도넛 차트 + 리스트
- 각 자산군이 **올해 순자산 변동에 얼마나 기여했는가** (%, 원)
- 예: "주식 +1,200만 (기여 +3.2%) / 부동산 -500만 (-1.3%) / 현금 저축 +800만 (+2.1%)"
- 클릭 시 해당 자산군 상세

### 4.4 현금흐름 화면

**기간 선택**: 이번달 / 올해 / 작년 / 임의 구간

**상단**: 수입 / 지출 / 저축률 (= (수입 - 지출) / 수입)

**중단**: 카테고리별 막대 차트 (지출 비중)

**하단**: 거래 리스트 (날짜 역순, 카테고리 필터 가능)

### 4.5 자산 상세 화면
- 자산명, 카테고리, 현재 평가액
  - **외화 자산일 경우**: 본 통화 금액을 메인 폰트로, 그 아래 KRW 환산값을 보조 폰트로 표시 (예: `$5,000` / `≈ 6,900,000원, 1 USD = 1,380원`)
- 스냅샷 히스토리 (시계열 그래프 + 리스트)
  - 그래프는 토글로 "본 통화" / "KRW 환산" 전환 가능
- "스냅샷 추가" 버튼 (날짜 + 금액 + 메모, 외화면 환율 입력 필드 추가)
- 관련 거래 (해당 자산이 fromAsset 또는 toAsset 인 거래)

### 4.6 가구 워크스페이스 & 멤버 권한

**워크스페이스 모델**
- 사용자는 **자신의 토스 계정으로 로그인**(appLogin) 후, 본인이 소유하거나 초대받은 **가구 워크스페이스(Household)** 에 진입
- 한 사용자는 여러 가구에 소속 가능 (본인 가구 + 부모님 가구 등). 화면 상단에서 워크스페이스 전환
- 모든 자산·거래·스냅샷·정기지출은 워크스페이스 단위로 격리

**권한 (Role)**
| 권한 | 자산 등록/수정 | 거래 등록/수정 | 정기지출 관리 | 멤버 초대/제거 | 조회 |
|---|:-:|:-:|:-:|:-:|:-:|
| **OWNER** | O | O | O | O | O |
| **EDITOR** | O | O | O | X | O |
| **VIEWER** | X | X | X | X | O |

- OWNER 는 워크스페이스를 만든 사람 (1명). OWNER 권한 양도 가능.
- EDITOR 는 무제한 추가 가능 (배우자 등 입력에 함께 참여하는 멤버).
- VIEWER 는 무제한 추가 가능 (자녀, 부모, 회계사 등 조회만 필요한 멤버).
- VIEWER 에게는 거래 입력 화면, FAB 버튼 등 편집 UI가 노출되지 않음.

**초대 플로우**
1. OWNER 가 "멤버 초대" 화면에서 권한(EDITOR/VIEWER) 선택
2. 초대 링크 또는 코드 생성 → 토스 메시지/카카오톡 등으로 공유
3. 초대받은 사람이 본인 토스 계정으로 로그인 → 초대 수락 → 해당 워크스페이스에 멤버로 추가
4. OWNER 는 언제든지 멤버 권한 변경 또는 제거 가능

**데이터 모델**
```
Household {
  id
  name              // "우리집"
  ownerUserId
  createdAt
}

Membership {
  id
  householdId
  userId
  role              // OWNER | EDITOR | VIEWER
  invitedAt
  joinedAt
  invitedByUserId
}

Invitation {
  id
  householdId
  role
  code              // 초대 코드/토큰
  expiresAt
  createdByUserId
  acceptedByUserId  // 수락 시 채워짐
}
```

> **삭제된 항목**: 기존 "1가구 1계정 (자격 증명 공유)" 모델은 폐기. 이제 각 사용자는 본인 토스 계정으로 로그인하고, 워크스페이스 멤버십으로 권한이 부여됩니다. "개인+공동 자산 분리"는 본 앱의 범위가 아니므로 Phase 2 후보에서도 제외 (개인 자산은 사용자가 별도로 관리).

## 5. 화면 구성 (Apps-in-Toss Mini App)

### 5.1 정보 구조
```
[상단 헤더] 워크스페이스 전환 (여러 가구 소속 시 드롭다운)

[탭 바: 4개]
1. 홈 (대시보드)        — 4.3
2. 자산                 — 자산 목록 → 자산 상세 (4.5)
3. 가계부               — 거래 입력/목록 + 정기지출 관리 (4.4, 4.2.1)
4. 더보기               — 멤버 관리, 카테고리 관리, 데이터 내보내기, 설정
```

> VIEWER 권한 멤버에게는 모든 화면의 입력/편집 UI(추가 버튼, 수정 아이콘 등)가 비활성화 또는 숨김 처리됩니다.

### 5.2 핵심 플로우

**Flow A: 월말 자산 스냅샷**
1. 홈 → "이번 달 스냅샷 업데이트" 카드 (마지막 입력 후 30일 경과 시 노출)
2. 자산 리스트 화면, 각 자산에 "최근 평가액" + 입력 박스
3. 한꺼번에 여러 자산 평가액 입력 → 저장 → 대시보드 즉시 갱신

**Flow B: 거래 기록 (공과금 예시)**
1. 가계부 탭 → FAB "+"
2. 지출 / 금액 / 카테고리(주거-공과금) / 출금 자산 선택
3. 제목 "1월 전기세", 메모 "사용량 320kWh"
4. 저장

**Flow C: 작년 대비 분석**
1. 홈 상단 "전년 대비 +12.4%" 탭
2. 비교 화면: 작년 동월 vs 이번 달 자산 구성, 자산군별 증감 워터폴 차트

**Flow D: 정기지출 등록 (예: 넷플릭스 구독)**
1. 가계부 탭 → 상단 세그먼트에서 "정기지출" 선택
2. "+" → 매월 / 17일 / 13,500원 / 카테고리(구독) / 출금 자산 선택
3. 제목 "넷플릭스", 적용 시작일 설정 → 저장
4. 매월 17일 자동으로 거래 1건 생성, 가계부 목록에 "자동" 뱃지로 표시

**Flow E: 가족 초대 (예: 부모님께 뷰어 권한)**
1. 더보기 → 멤버 관리 → "초대하기"
2. 권한 선택 (EDITOR / VIEWER)
3. 초대 코드/링크 발급 → 토스 메시지로 공유
4. 초대받은 사람이 본인 토스 계정으로 로그인 → 수락 → 워크스페이스 멤버 추가

### 5.3 디자인 톤
- **TDS React Native** 컴포넌트 사용 (`@toss/tds-react-native`)
- 토스 디자인 시스템에 맞춘 모노톤 + 강조 컬러 1색
- 숫자 강조 (큰 폰트, 천단위 콤마, 음수 빨강/양수 파랑은 토스 컨벤션 따름)
- 입력은 **숫자 키패드 우선**, 자산 선택은 시트(BottomSheet)

## 10. 참고: 데이터 모델 요약 (ERD 수준)

```
User ─── Membership ─── Household
                         ├─ Asset (1:N)
                         │     └─ AssetSnapshot (1:N)
                         ├─ Transaction (1:N)
                         │     ├─ fromAsset (N:1, nullable)
                         │     ├─ toAsset (N:1, nullable)
                         │     └─ recurringTemplateId (N:1, nullable)
                         ├─ RecurringTransaction (1:N)
                         ├─ Invitation (1:N)
                         └─ Category (1:N)

Membership { householdId, userId, role: OWNER|EDITOR|VIEWER }
```

- User ─ Household 는 **N:M (Membership 통해)**: 한 사용자가 여러 가구에, 한 가구는 여러 멤버에 연결.
- Transaction 의 `recurringTemplateId` 는 자동 생성된 거래의 출처 추적용 (수정 후에도 어떤 정기지출 템플릿에서 나왔는지 표시).
- 본 앱은 가구 공동 자산만 다루므로 자산/거래에 `scope` 필드는 두지 않습니다.
- `Asset.currency` + `AssetSnapshot.fxRateToKRW` + `AssetSnapshot.valueKRW` 조합으로 다통화를 지원합니다. 합산·대시보드 계산은 모두 `valueKRW` 기준이며, 화면 표기 시에는 본 통화도 함께 노출합니다.
