# 라오어 자동매매 자동화 웹앱 — Claude Design 핸드오프 v2

> **용도:** 이 문서는 Claude Design이 UI/UX를 설계하기 위한 단일 레퍼런스다.
> 무한매수법(무매) + 밸류리밸런싱(VR) 두 전략의 비즈니스 로직·공식·화면 구조·데이터 모델이 모두 담겨 있다.
> 디자인 결과물이 이 문서를 완전히 커버해야 핸드오프 성공이다.

---

## 1. 프로젝트 개요

### 무엇을 만드는가

"라오어 무한매수법 V4.0" 과 "밸류리밸런싱 VR5.0" 운용을 자동화하는 **통합 자동매매 대시보드 웹앱**이다.

두 전략 모두 TQQQ/SOXL (미국 3배 레버리지 ETF)을 대상으로 매일 LOC(Limit On Close) 주문으로
분할 매수·매도하는 장기 전략이다. 사용자는 매일 오늘 주문 가격과 수량을 확인하고, 증권사 앱에서
직접 주문을 넣는다. 장 마감 후 체결된 내역을 앱에 입력하면 시스템이 상태값을 자동 갱신하고
내일 계획을 다시 계산해준다.

### 현재 상황 (자동화 전)

- 상태파일이 `.md` 파일로 로컬에 존재
- 매일 사용자가 체결 결과를 Claude에게 자유서술로 보고 → Claude가 계산·갱신
- 사람이 직접 계산하는 구조라 실수 가능성 존재, 모바일 접근 불가

### 목표

| 지금 | 자동화 후 |
|---|---|
| Claude에게 자유서술 보고 | 웹앱 UI에서 폼으로 체결 입력 |
| Claude가 수동 계산·MD 갱신 | 서버가 자동 계산·DB 저장 |
| 로컬 파일, 단일 기기 | 자체 서버, 다중 기기 접근 |

### 사용자 페르소나

- **주 사용자**: 라오어 무매/VR을 실제로 운용 중인 개인 투자자 (본인)
- **기술 수준**: 비개발자. 증권사 앱에 익숙한 수준
- **사용 패턴**: 하루 2회 접속 — 아침 (오늘 주문 확인), 저녁 장 마감 후 (체결 입력)
- **핵심 니즈**: 계산 실수 없이 오늘 LOC 주문 가격·수량을 빠르게 확인

## 4. 무한매수법 — 도메인·공식·모드

### 4-1. 핵심 용어 정의

#### T값 (Turn)

진행 회차를 나타내는 소수점 숫자.

```
1회 매수 체결  → T += 1
절반 매수 체결 → T += 0.5
쿼터매도 체결  → T = T × 0.75
지정가매도 후 LOC 매수 1회 체결  → T = T × 0.25 + 1
지정가매도 후 LOC 매수 절반 체결 → T = T × 0.25 + 0.5
```

#### 평단 (평균단가)

보유 주식의 평균 매입가. 매도 시 변동 없음.

```
매수 시: 평단_new = (평단_old × 수량_old + 체결가 × 체결수량) / (수량_old + 체결수량)
```

#### 잔금

원금 중 아직 매수에 사용하지 않은 금액.

#### 별% (★%)

별지점을 계산하는 기준 퍼센트. T가 커질수록 감소하며, 후반전에서 음수가 된다.

```
TQQQ 40분할: 별% = (15 - 0.75 × T) %
SOXL  40분할: 별% = (20 - T) %
```

#### 별지점 (★지점)

매수와 매도를 가르는 기준 가격.

```
별지점   = 평단 × (1 + 별%)
LOC 매수가 = 별지점 - 0.01   ← 별지점 바로 아래
LOC 매도가 = 별지점           ← 그대로
```

#### LOC (Limit On Close)

장 마감 시 지정가에 도달하면 체결하는 주문 유형.

#### 1회 매수액

```
1회 매수액 = 잔금 / (분할수 - T)
```

#### 쿼터매도

```
쿼터매도 수량 = floor(보유수량 / 4)
```

### 4-2. 모드 분류

| 모드명 | 조건 | 매수 구조 | 매도 구조 |
|---|---|---|---|
| 사이클시작 | T = 0 | 큰수매수. 전날종가 ×1.10 LOC | 없음 |
| 일반-전반전 | 0 < T < 20 | 1회매수액 절반 → 별지점LOC + 절반 → 평단LOC | 쿼터매도(별지점) + 15%/20% 지정가 |
| 일반-후반전 | 20 ≤ T < 40 | 1회매수액 전체 → 별지점LOC (별지점 < 평단) | 동일 |
| 리버스모드 | T > 39 | 잔금/4 쿼터매수 (별지점 LOC) | 첫날 MOC, 이후 LOC |

### 4-3. 모드 상태 전이

```
[새 사이클]
T = 0
    │
    ↓ [사이클시작] ── 큰수매수 LOC ──→ 체결 시 T ≥ 1
    │
    ↓ [일반-전반전] 0 < T < 20
    │    별지점LOC + 평단LOC
    │
    ↓ T ≥ 20
    │
    ↓ [일반-후반전] 20 ≤ T < 40
    │    별지점LOC 전액
    │
    ↓ T > 39
    │
    ↓ [리버스모드]
    │    별지점 = 5거래일 종가 평균
    │    첫날 MOC 매도 / 이후 LOC 매도+쿼터매수
    │
    ↓ 종가 > 평단 × 0.85 (TQQQ)
    │
    └→ [일반-전반전] 복귀 (T값 그대로)

별도 분기: 보유수량 = 0
    → 사이클 종료 → 복리/단리 선택 → T=0 리셋
```

### 4-4. 계산 공식 전체

#### 4-4-1. 일반-전반전 매수

```
1회 매수액    = 잔금 / (분할수 - T)
별지점        = 평단 × (1 + 별%)
LOC 매수가_별 = 별지점 - 0.01
LOC 매수가_평 = 평단

절반 금액   = 1회 매수액 / 2
별지점 수량 = floor(절반 금액 / LOC 매수가_별)   → 0이면 1주로 올림
평단 수량   = floor(절반 금액 / LOC 매수가_평)   → 0이면 1주로 올림

※ 별지점 1주 + 평단 1주 합계가 1회매수액 초과해도 허용 (방법론 취지)
```

#### 4-4-2. 일반-후반전 매수

```
별지점 수량 = floor(1회 매수액 / LOC 매수가_별)
※ 별지점 < 평단 (별%가 음수이므로)
```

#### 4-4-3. 사이클시작 매수 (T=0)

```
큰수 LOC 가격 = 전날 종가 × 1.10
큰수 수량     = floor(1회 매수액 / 큰수 LOC 가격)
분할 LOC는 큰수 가격 아래로 매 단계 약 7% 할인씩, 1주씩
```

#### 4-4-4. 매도 (전반전/후반전 공통)

```
쿼터매도가    = 별지점              (LOC 매도)
쿼터매도 수량 = floor(보유수량 / 4)

지정가 매도가 = 평단 × 1.15        (TQQQ, 15%)
지정가 매도가 = 평단 × 1.20        (SOXL,  20%)
지정가 수량   = 보유수량 - 쿼터매도 수량
```

#### 4-4-5. 리버스모드 별지점

```
리버스 별지점 = 직전 5거래일 종가의 평균
LOC 매도가    = 리버스 별지점
LOC 매수가    = 리버스 별지점 - 0.01
```

#### 4-4-6. 리버스모드 매도

```
첫날 매도: floor(보유수량 / 20)    (MOC)
이후 매도: floor(직전 보유수량 / 20)  (LOC, 매일)
```

#### 4-4-7. 리버스모드 쿼터매수

```
매수 금액 = 잔금 / 4
매수 수량 = floor(매수 금액 / LOC 매수가)
```

#### 4-4-8. 리버스모드 T값 갱신

```
매도 후: T = T × 0.95
매수 후: T = T + (분할수 - T) × 0.25
```

#### 4-4-9. 리버스모드 종료 조건

```
TQQQ: 종가 > 평단 × 0.85   → 일반모드 복귀
SOXL: 종가 > 평단 × 0.80   → 일반모드 복귀
```

#### 4-4-10. 분할 LOC 가격 (추가 매수라인)

```
가격_n = 가격_(n-1) × (1 - 0.07)   ← 약 7% 할인씩, 1주씩
```

### 4-5. 리버스모드 상세 플로우

**진입**: T > 39 (40분할 기준)

**첫날**
- 매도만: floor(보유수량 / 20) 개를 MOC 매도
- 매수 없음
- T = T × 0.95

**둘째날 이후 (매일 반복)**

1. 별지점 계산: 직전 5거래일 종가 평균
2. 매도: floor(직전 보유수량 / 20) 개를 리버스 별지점 LOC
3. 매수: 잔금 / 4 금액으로 리버스 별지점 - 0.01 LOC
4. 종료 판단: 종가 > 평단 × 0.85 (TQQQ) → 일반모드 복귀

## 5. VR(밸류리밸런싱) — 도메인·공식·운용

### 5-1. 핵심 개념

- 책 "밸류리밸런싱"(2022, 라오어). VR5.0 = 기본공식 편
- 목적: TQQQ 무지성 매수 대비 **MDD 절감(QLD 수준)**, 수익률은 거의 동일 유지
- 백테스트 결과: P/V/G=10 = 수익률 2.98× / 위험도 2.37× (TQQQ는 3×/3×, QLD는 2×/2×)
- 평균 현금비중 13.2%를 유지하면서도 수익률 손실 최소화

### 5-2. 변수 정의

| 변수 | 의미 |
|---|---|
| **V** (Value) | TQQQ 평가금이 가야 할 가이드 수치. 최소·최대 밴드를 결정 |
| **Pool** | 보유 현금 |
| **G** (gradient) | 기울기 결정 요소. 클수록 V 기울기 낮음 → 안정적. 적립/거치=10, 인출=20 권장 |
| **밴드** | V × 0.85 ~ V × 1.15 (±15%, ±10%, ±5% 선택 가능) |
| **평가금** | 보유수량 × 현재가 |
| **운용 방식** | 적립식(사이클마다 Pool에 추가) / 거치식(추가 없음) / 인출식(사이클마다 인출) |

### 5-3. 핵심 공식

```
V₂ = V₁ + Pool/G ± (적립금 or 인출금)

매수 발동 조건: 평가금 < V × (1 - 밴드%)   ← 예: V × 0.85
매도 발동 조건: 평가금 > V × (1 + 밴드%)   ← 예: V × 1.15
```

**계산 예시** (적립식, G=10, 매사이클 $250 적립):
```
현재: V₁ = $9,000 / Pool = $1,000
V₂ = $9,000 + $1,000/10 + $250 = $9,350
```

### 5-4. Pool 사용 제한 (한 사이클당)

| 운용 방식 | Pool 사용 한도 |
|---|---|
| 적립식 | 적립 후 Pool × **75%** |
| 거치식 | Pool × **50%** |
| 인출식 | 인출 후 Pool × **25%** |

가이드일 뿐이며 사용자가 조정 가능 (많이 사용할수록 공격적).

### 5-5. 사이클 = 2주

- 매 사이클 시작 시 V 갱신 공식으로 V₂ 계산
- 새 밴드(V₂ × 0.85, V₂ × 1.15)로 매수매도표 재계산
- 기준 요일 자유 (라오어: 월~다음주 금)

### 5-6. 매수매도표 구조 (계단식 LOC)

VR은 밴드 이탈 시에만 체결이 발생한다. 계단식으로 1주씩 분할 LOC.

**매수표** (평가금 < 최소밴드 시 발동):

| 최소값 (V×0.85) | 매수 후 수량 | 매수점(LOC가격) | 매수 후 Pool |
|---|---|---|---|
| **현재 최소밴드** | **현재수량** | - | **현재 Pool** |
| | 현재수량+1 | 매수점₁ | Pool - 매수점₁ |
| | 현재수량+2 | 매수점₂ | Pool - 매수점₁ - 매수점₂ |
| | ··· | ··· | ··· |

각 행은 "이 가격에서 1주 매수 체결 시의 상태". Pool이 한도(75%) 이하로 내려가면 표 종료.

**매도표** (평가금 > 최대밴드 시 발동): 동일 구조, 최대밴드 기준.

### 5-7. VR 엣지 케이스

- **Pool 마이너스 금지**: Pool 소진 시 추가 매수 불가. 외부 자금 투입 X
- **발동 없는 날**: 평가금이 밴드 안에 있으면 당일 주문 없음 (홀드)
- **수익률 함정**: 적립식 계좌는 수익률이 실제보다 낮아 보임. 입금원금/투자원금 구분 필요
- **G값 점진 변경**: 54주차부터 /11, 1년마다 /12, /13... (설정 메모 제공)

## 6. 통합 추상 레이어

두 전략을 하나의 앱에서 처리하기 위한 전략 인터페이스.

### 6-1. Strategy 인터페이스

```typescript
interface Strategy {
  computeDailyPlan(state: StrategyState, closingPrice: number): DailyPlan;
  // → {buy_rows: PlanRow[], sell_rows: PlanRow[], triggered: boolean}

  applyFills(state: StrategyState, fills: Fill[]): StrategyState;
  // → T·평단·잔금 or V·Pool 갱신

  checkCycleEnd(state: StrategyState): CycleEndSignal | null;
  // → 무매: 보유=0 / VR: 2주 경과 또는 null
}
```

### 6-2. 구현체

| 구현체 | 전략 | 특이사항 |
|---|---|---|
| `InfiniteBuyingStrategy` | 무매 | 모드 분기(4종), 큰수매수, 리버스, T값 갱신 |
| `ValueRebalancingStrategy` | VR | 밴드 판정, Pool 한도 적용, V 갱신(2주) |

### 6-3. 공통 유틸리티

```typescript

computeAvgPrice(oldAvg, oldQty, execPrice, execQty): number

  → POST /strategies/:id/executions
  → 백엔드: 상태 갱신 → daily_plan 재계산
  → 응답: {new_state, tomorrow_plan}
  → Bottom Sheet 닫힘 → 카드 즉시 업데이트
```

## 13. 전략 상세 페이지

```
[← 대시보드]   TQQQ 상세   [무매/VR 배지]

[히스토리] [차트] [타임라인]
```

### 13-1. 히스토리 탭

**무매 컬럼**:
| 날짜 | 구분 | 가격 | 수량 | 체결금액 | T 변화 | 변화 후 T | 변화 후 평단 | 변화 후 잔금 |

**VR 컬럼**:
| 날짜 | 구분 | 가격 | 수량 | 체결금액 | Pool 변화 | 변화 후 Pool | 변화 후 수량 |

최신 체결이 최상단.

### 13-2. 차트 탭

**무매**:
- T값 추이 (선 그래프, 날짜 x축)
- 평단 vs 종가 추이 (이중선)
- 별지점 밴드 표시 (±별% 영역)

**VR**:
- V 밴드 (최소~최대, 배경 색상 구역)
- 평가금 추이 (선 그래프)
- Pool 추이 (하단 바 차트)

### 13-3. 타임라인 탭

**무매**:
```
[사이클시작] 2026-05-06
    ↓
[일반-전반전] 2026-05-07 ~ 현재 (T=4.5)
    ↓ (예정: T≥20)
[일반-후반전]
```

**VR**:
```
[사이클 1] 2026-05-06 ~ 05-19  V₁=$8,700 → V₂=$9,000
    ↓
[사이클 2] 2026-05-20 ~ 06-02  V₁=$9,000 → (진행 중)
```

## 14. 시세 API

### 아키텍처

클라이언트는 외부 API 직접 호출 안 함. NestJS가 프록시+캐시.

```
프론트 → GET /prices/:ticker/latest → PricesModule → prices 테이블
                ↓ 캐시 없거나 만료
           외부 API fetch
```

### Cron (자동 fetch)

```typescript
@Cron('30 16 * * 1-5', { timeZone: 'America/New_York' })
// 평일 ET 16:30, 무매·VR 모두 사용하는 TQQQ, SOXL 종가 fetch
```

### 추천 소스

- `yahoo-finance2` npm (무료, 안정적)
- Alpha Vantage 무료 tier (하루 25회 제한, fallback용)

### 수동 fallback

- UI에서 "시세 새로고침" 버튼 → `POST /prices/:ticker/refresh`
- 자동 fetch 실패 시 사용자가 직접 종가 입력 가능 (체결 입력 Bottom Sheet 상단)

## 15. 엣지 케이스

### 무매 — 큰수매수

**발동**: 별지점과 종가 괴리 ≥ 15%

**UI 처리**:
- 매수표 위 경고 배너
- 큰수 LOC 가격 자동 제안: `종가 × 1.15` (사용자 조정 가능)
- 해당 가격 위 모든 수량 합산 → 큰수 행에 표시

**원칙**: 낮은 가격에서 매수 안 걸리는 것이 문제. 별지점 아래에서 무조건 매수 필요.

### 무매 — 매도거부

**발동**: 큰 하락 시 별지점 매도가와 현재가 괴리 과대

**UI 처리**:
- 매도표에 안내: "종가 괴리 과대 시 미체결 가능. 낮은 가격 변경 절대 금지."
- 체결 입력 구분에 "미체결(매도거부)" 선택지

### 무매 — LOC 정수 단위

- 최소 1주 (정수만)
- 절반 수량 결과가 0이면 1주로 올림
- 초과분은 다음날 `잔금/(분할수-T)` 재계산으로 흡수

### 무매 — 사이클 종료

```
if 보유수량 == 0:
    → 사이클 종료 오버레이
    → 복리: 새 원금 = 잔금 (수익 포함)
    → 단리: 새 원금 = 최초 설정 원금
    → strategies.cycle_no += 1
    → state 초기화 (T=0, 평단=null, 보유=0)
```

### VR — Pool 소진

- Pool < 0 불가. 매수 버튼 비활성화
- UI에 "Pool 부족 — 사이클 갱신 후 적립금 확인" 안내

### VR — 밴드 내 홀드

- 평가금이 밴드 안에 있으면 오늘 주문 없음
- VR 카드에 "오늘 주문 없음 (밴드 내)" 상태 명시
- 체결 입력 Bottom Sheet는 "미체결" 선택지만 제공

### VR — 수익률 함정 안내

- 설정 화면에 소정의 안내: "적립식은 계좌 수익률이 실제보다 낮아 보일 수 있습니다."
- 정확한 수익률은 입금원금 기준으로 별도 계산 필요

## 16. 시각 디자인 톤 (토스 스타일)

### 16-1. 전반적 톤

- **라이트 모드 기본** (다크는 옵션으로 제공 가능)
- 한 화면 한 핵심 메시지. 군더더기 없는 레이아웃
- 카드: 흰 배경 + 16px radius + `box-shadow: 0 2px 8px rgba(0,0,0,0.06)`
- 색상은 절제. 숫자와 CTA 버튼에만 강조색
- 숫자 변경 시 미세한 fade transition만

### 16-2. 컬러 시스템

| 의미 | 색상 | 비고 |
|---|---|---|
| 주요 액션 (CTA) | **#3182f6** (Toss Blue) | 저장 버튼, 확인 버튼 |
| 상승·매수 | **#f04452** (Toss Red) | 한국 주식 관례: 상승=빨강 |
| 하락·매도 | **#2563eb** (Blue-600) | 한국 주식 관례: 하락=파랑 |
| 별지점·강조 | **#f59e0b** (Amber-500) | ★ 행 배경 |
| 경고 (리버스/큰수매수) | **#ff7733** (Toss Orange) | 배너 배경 |
| 배경 | **#f9fafb** | 전체 페이지 배경 |
| 카드 | **#ffffff** | 카드 표면 |
| 본문 텍스트 | **#191f28** | 주요 숫자, 라벨 |
| 보조 텍스트 | **#6b7684** | 섹션 헤더, 캡션 |
| 구분선 | **#f2f4f6** | 1px |

### 16-3. 타이포그래피

- **폰트**: Pretendard (한국 금융앱 표준, 웹폰트 로드)
- **숫자 (가격, T값, V값 등)**: `font-variant-numeric: tabular-nums` — 자릿수 흔들림 방지
- **카드 메인 숫자 (평가금, 평단 등)**: 24~28px Bold
- **섹션 라벨**: 12px, #6b7684, `letter-spacing: 0.04em`
- **본문 / 표 셀**: 16px Regular
- **CTA 버튼 텍스트**: 17px Medium

### 16-4. 표 형식 유지 원칙

방법론 원문의 3컬럼 표 (구분 / 가격 / 수량) 를 그대로 유지한다.
디자이너 임의로 카드형·슬라이더형 등으로 변환 금지. 모바일 가로 스크롤 허용.

```
< 매수점 >
| 구분   | LOC 가격 | 수량 |    ← 헤더: 12px 회색 캡션
|--------|---------|------|
| ★ LOC  | $83.82  |  1   |    ← amber 배경
| 평단   | $75.10  |  1   |    ← 연한 강조 배경
| 분할③  | $69.84  |  1   |    ← 기본
```

### 16-5. 표 행 색상

| 행 종류 | 색상 |
|---|---|
| ★ LOC 매수 / ★% LOC 매도 | amber 배경 (#fffbeb) + amber 텍스트 |
| 평단 LOC | 연한 파랑 배경 (#eff6ff) |
| 분할 매수 / 일반 VR 매수 | 기본 흰 배경 |
| 쿼터매도 / VR 매도 | 연한 빨강 (#fff1f2) |

### 16-6. 모바일 가이드라인 (필수)

**터치 타겟**
- 모든 버튼·탭·아이콘: 최소 **44×44pt** (iOS HIG)
- CTA 버튼: 높이 **56dp**, 전체 너비, radius 14px

**간격·여백**
- 카드 내부 패딩: **16px**
- 카드 간 간격: **12px**
- 표 행 높이: 최소 **44px**
- 섹션 간 구분선: 1px #f2f4f6

**Safe Area 대응**
- `env(safe-area-inset-top)`, `env(safe-area-inset-bottom)` 반드시 적용
- Bottom Tab Bar: 하단 safe area 위에 고정
- Bottom Sheet CTA: 홈 인디케이터 위로 패딩 확보

**기타**
- 표 컨테이너: `overflow-x: auto` (페이지 전체 가로 스크롤 금지)
- Pull-to-refresh: 대시보드 → 시세 갱신 트리거
- 숫자 입력: iOS 숫자 키패드 (`inputmode="decimal"`)
